WordCanvas API Documentation

Endpoints

Authentication

  • POST /api/auth/signup

    Description: Register a new user.

    Required Data:

    • name: String
    • email: String
    • password: String

    Success Response:

    { "success": true, "data": { "id": "123", "name": "John Doe" } }

    Error Response:

    { "success": false, "error": "Email already exists." }
  • POST /api/auth/login

    Description: Log in an existing user.

    Required Data:

    • email: String
    • password: String

    Success Response:

    { "success": true, "data": { "token": "JWT-TOKEN", "user": { "id": "123", "name": "John Doe" } } }

    Error Response:

    { "success": false, "error": "Invalid credentials." }

Blog Management

  • GET /api/blogs

    Description: Fetch all blog posts.

    Optional Query Parameters:

    • author: Filter by author ID

    Success Response:

    { "success": true, "data": [ { "id": "1", "title": "First Blog", "author": "John Doe" } ] }

    Error Response:

    { "success": false, "error": "Server error." }
  • POST /api/blogs

    Description: Create a new blog post.

    Required Data:

    • title: String
    • content: String
    • author: User ID (from JWT token)

    Success Response:

    { "success": true, "data": { "id": "1", "title": "First Blog" } }

    Error Response:

    { "success": false, "error": "Unauthorized access." }

General Response Structure

  • success: Boolean indicating if the request was successful.
  • data: The requested data, if available.
  • error: Error message, if any.
